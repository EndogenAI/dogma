# Workplan: Sprint 23 — Recommendation Provenance

**Branch**: `feat/recommendation-provenance-sprint`
**Date**: 2026-03-20
**Orchestrator**: Executive Orchestrator

---

## Objective

Build a complete recommendation provenance system for the dogma repository: define a machine-readable `recommendations:` frontmatter schema for D4 synthesis docs, enforce it in CI, automate index generation, run a semi-automated retrofit audit across all 28 finalized synthesis docs, apply human-verified patches, and hook the indexer into the on-archival and weekly CI triggers. This closes the broken link between `docs/research/` recommendations and the GitHub issues they generated. Substrate Distiller consolidation (#403, #404) is a stretch goal if sprint capacity allows after Phase 7.

---

## Design Decisions

| # | Decision | Detail |
|---|----------|--------|
| 1 | **Dual-format schema** | `recommendations:` YAML block in frontmatter (machine/agent-readable) + inline status in `## Recommendations` text (human-readable) |
| 2 | **7-state status taxonomy** | `accepted`, `accepted-for-adoption`, `adopted`, `completed`, `rejected`, `not-accepted`, `deferred`; each entry: `id`, `status`, `linked_issue`, optional `decision_ref` |
| 3 | **Registry** | `data/recommendations-registry.yml` — auto-generated by `scripts/index_recommendations.py`, tracked in git |
| 4 | **CI hard-fail** | `validate_synthesis.py` fails for any `status: Final` synthesis doc missing a `recommendations:` block; warning for non-synthesis finalized docs |
| 5 | **Retrofit scope** | 28 `status: Final` synthesis docs only; OPEN_RESEARCH.md excluded; Phase 5 is a mandatory human review gate before any doc is modified |
