# Workplan: Q2 Wave 2 Phase 3–4

## Objective

Phase 3 standardizes the agent fleet by auditing and correcting tool-count ceiling
violations, encoding decision tables and guardrail constraints as schema-validated YAML,
implementing multi-provider inference abstraction, and shipping an Adoption agent for
companion-repo onboarding. Phase 4 adds an observability and metrics layer: OTel
instrumentation for all LLM spans, /health endpoints, session schema extensions for
Telemetry and Audit Trail sections, phase-gate encoding, and slow-error-discovery
detection scripts — closing the feedback loop between agent behavior and measurable
outcomes.
