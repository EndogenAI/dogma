## Critical Path & Dependencies

**Research-First Constraint**: All 3 retrospective/research phases (1–3) must complete and pass Review gates before Phase 4 (Primary Research Protocol docs) begins. This ensures Phase 4 docs are informed by the research findings.

**Batch Constraint (Programmatic-First)**: Phases 5 and 6 are execution phases. Phase 5 (inventory script) must run first to identify the docs that Phase 6 (patches) will operate on. This prevents blind patching and enforces script-gating over manual execution.

**Parallelisation Opportunity**: Research phases 1–3 can run in parallel if researcher capacity exists; each still requires its own Review gate before proceeding to Phase 4.
