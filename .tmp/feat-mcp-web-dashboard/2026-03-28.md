### **Gap 1+2 (BLOCKING): MCP Inspector architecture, extensibility, license**

Source: https://github.com/modelcontextprotocol/inspector

- **Architecture**: Two-component system:
  - **MCPI (MCP Inspector Client)**: React-based web UI for interactive testing/debugging
  - **MCPP (MCP Proxy)**: Node.js server acting as protocol bridge — connects web UI to MCP servers via stdio/SSE/streamable-http transports
  - Proxy is NOT a network proxy — it's both an MCP client (to your server) and HTTP server (serving the UI)
- **Extensibility**:
  - Supports 3 transport types: stdio (default), SSE, streamable-http
  - Config file support (`mcp.json`) for multi-server setups
  - CLI mode for programmatic interaction (scripting, CI/CD)
  - Data source plugins hook into existing data via APIs
- **License**: MIT (confirmed from LICENSE file mention in README)
- **Build decision**: **Build-alongside** — MCP Inspector is a developer tool for testing/debugging, NOT a production dashboard framework. dogmaMCP needs a read-only observability UI; Inspector is interactive/write-capable.
- **Key takeaway**: Inspector's proxy architecture (protocol bridge pattern) is reusable, but the UI is overfit for interactive testing (forms, sampling, tool invocation). dogmaMCP needs metrics-focused visualization, not full MCP client capabilities.

### **Gap 4 (BLOCKING): FastAPI SSE + Svelte EventSource integration**

**FastAPI SSE pattern** (https://fastapi.tiangolo.com/advanced/custom-response/#streamingresponse):
- Use `StreamingResponse` with async generator:
  ```python
  async def event_stream():
      for i in range(10):
          yield b"some fake data"
          await anyio.sleep(0)  # allow cancellation
  return StreamingResponse(event_stream())
  ```
- CORS: Must add `CORSMiddleware` for cross-origin requests
- Format: SSE requires `data: <payload>\n\n` format (not documented in snippet, but standard SSE protocol)

**Svelte EventSource**:
- Standard browser `EventSource` API (no library needed):
  ```js
  const eventSource = new EventSource('http://localhost:8080/sse');
  eventSource.onmessage = (event) => { console.log(event.data); };
  ```
- Reconnect: Automatic (browser handles it)
- Error handling: `onerror` event fires on connection failure

**Canonical integration**:
1. FastAPI: `StreamingResponse(async_generator(), media_type="text/event-stream")`
2. Svelte: `new EventSource(<url>)` in component `onMount`
3. CORS: Add `allow_origins=["http://localhost:5173"]` to FastAPI CORS config

**Missing**: No official "FastAPI + Svelte SSE" tutorial found — pattern is generic (works with any SSE client). CORS is the most common footgun.

### Draft Rec 3: SSE + Staleness UX

**FastAPI SSE endpoint** (`GET /events`):
- `StreamingResponse(async_generator(), media_type="text/event-stream")`
- Emit `data: <json>\n\n` on each new tool invocation
- Emit keepalive comment (`: keepalive\n\n`) every 15s to prevent proxy timeout
- CORS footgun: `CORSMiddleware` with `allow_origins=["http://localhost:5173"]` required

**Svelte EventSource component**:
- Standard browser `EventSource` API — no extra library needed
- Subscribe in `onMount`, close in `onDestroy` (prevent listener leaks)
- Append-only bounded array (cap at last N=50 events)
- On `onerror`: set `ERROR` state, exponential backoff retry (2s → 4s → 8s, max 30s)

**Live/Stale state machine**:
```
CONNECTING → (open) → LIVE (green dot)
LIVE → (onerror/onclose) → STALE (amber dot + "Last updated X min ago")
STALE → (reconnect) → CONNECTING
STALE → (REST snapshot fetch) → STALE (timestamp refreshes, dot stays amber)
```

**Coexistence rule**: SSE drives real-time invocation feed (append-only); REST `GET /snapshot` drives Overview tab metrics (user-triggered or 30s poll). Independent data paths — do not merge.

**Confidence**: HIGH — derivable from standard browser APIs + FastAPI docs; consistent with Grafana/Honeycomb conventions.

## Retrospective — 2026-03-28

_Phase 1 research → replanning cadence (Phases 1A–1J)_

### Lessons Learned

1. **Model-swap cadence yields higher trust**: A 10-step cadence (Sonnet 4.5 scouting / 4.6-HR synthesis) with human pause gates between substeps produced better directional alignment than a single monolithic delegation — each pause created a natural correction surface before the next model spent tokens.

2. **Skip-step optimization is valid and saves context**: When Gap Triage returns all decisions CLOSEABLE or DEFERRED_TO_IMPL, the tertiary scouting step (Phase 1F) can be skipped entirely. Encoding this as a rule saves one full model context per research sprint where the gap is exhausted.

3. **Commit research docs before sprint replanning**: Committing D4 docs as `Status: Final` before Phase 1I replanning means the planner reads locked final text and cites it precisely in Phase 2+ deliverables — avoiding "as described in research" vagueness.

4. **Executive Planner quality is input-format-sensitive**: The Planner produced the highest-precision amendments when given (a) a numbered list of exact locked decisions and (b) a concrete format mandate (`[Phase N — Section]: ADD/CHANGE "text" (reason: ...)`). Without the format mandate, output was structurally vague.

5. **Workplan Review gate overhead is low**: The Phase 0 review + one REQUEST CHANGES cycle cost only 1 extra token round-trip. The re-review cycle is not expensive when the fix is surgical (single missing row). Gate is worth the cost.

6. **Phase 1H issue-seeding is mandatory but unnamed**: Research Doc PR Merge Gate requires recommendations → tracked issues, but deep-research-sprint and session-management skills have no explicit "seed deferred-scope issues" step. Phase 1H fills this gap in practice; it should be named as a mandatory post-synthesis step in the skill.

7. **Pre-commit status vocab fires on commit, not creation** (already encoded in T3 gate; no new action needed).

8. **Library prohibition naming prevents familiarity-reach**: Naming the prohibited alternative explicitly ("do NOT install svelte-chartjs or chart.js") in Phase 4 deliverables is required to prevent implementers from defaulting to the heavier familiar library. Abstract "use LayerCake" is insufficient.

### Anti-patterns Avoided

1. Running replanning (Phase 1I) before D4 docs were committed — would have produced vague spec references.
2. Delegating a vague "update the workplan" prompt to Executive Planner — would have produced non-actionable diff.
3. Skipping Phase 0 workplan review and discovering ordering violations mid-implementation.

### Efficiency Observations

1. Phase 1F skip saved ~8,000–12,000 tokens (one Scout context).
2. Executive Planner 8-amendment output in one delegation vs. sequential per-section edits saved ~3–4 extra tool calls.
3. Pre-commit hook caught `status: open` in both docs on first commit attempt — would have slipped through manual review.
